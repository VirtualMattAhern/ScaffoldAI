2.) Locations within each RG, should attempt to either be all US East or all US West - or if needing to mix at all, mostly one or other + Central, but not mix of East and West (due to degredation).  

3.) Each product (BidBudAI-Prod, SkafoldAI-Prod, RezAGI-Prod) contain foundational resources:

Web container
API container
Database
Key Vault
Log Analytics
Storage
AI resource

4.) Domain mapping (custom domains per app):
   - SkafoldAI: www.skafoldai.com, api.skafoldai.com
   - RezAGI: www.rezagi.com, api.rezagi.com
   - BidBudAI: www.bidbudai.com, api.bidbudai.com

5.) ISOLATION: Each subscription and its resource group are completely separate. 
   - NO SkafoldAI resources or naming in BidBudAI-Prod / rg-bidbud-prod
   - NO BidBudAI resources or naming in SkafoldAI-Prod / rg-skafoldai-prod  
   - NO RezAGI resources or naming in either of the above
   Each application is fully isolated.

   See PRODUCTION_DEPLOYMENT.md for deployment steps (web, api, DNS, Entra ID).
